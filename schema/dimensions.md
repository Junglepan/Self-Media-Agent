# dimensions.md — 七维度定义与文件模板

> 角色由七个维度构成。本文件定义每一维度的**边界、文件归属、条目模板**。
> 提炼时若分不清某条信号属于哪一维度，参考本文件的"边界示例"。

---

## 七维度总览

| # | 维度 | 文件 | 一句话定义 |
|---|------|------|-----------|
| 1 | 人格 | `wiki/persona.md` | 其他六维的综合摘要（主入口） |
| 2 | 审美 | `wiki/aesthetics/` | 他喜欢什么样的画面 |
| 3 | 技能 | `wiki/skills/` | 他会用什么方式表达 |
| 4 | 学识 | `wiki/knowledge/` | 他知道什么、能联想到什么 |
| 5 | 世界观 | `wiki/worldview.md` | 他为什么拍、拍什么有价值 |
| 6 | 成长轨迹 | `wiki/growth.md` | 他在变成什么样 |
| 7 | 参照系 | `wiki/references.md` | 他站在谁的肩上、反对谁 |

---

## 1. 人格 — `wiki/persona.md`

**定义**：其他六维的综合摘要。不承载原始条目，只做汇总。

**何时更新**：每轮 Run Loop 的 Step 9。

**结构模板**：
```markdown
# Persona

> 一句话定位：<例如：一位偏爱城市边缘、低饱和、慢节奏的街拍摄影师>

## 人格画像
<3–5 句话，第三人称，像展牌>

## 六维摘要
### 审美
- <最高置信度 3–5 条，从 aesthetics/ 抽取>

### 技能
- ...

### 学识
- ...

### 世界观
- ...

### 成长
- ...

### 参照
- ...

## 当前正在演化的议题
- [待收敛] <条目标题> — <一句话说明分歧>
```

## 2. 审美 — `wiki/aesthetics/`

**定义**：对画面本身的偏好。**不涉及"为什么"，只涉及"长什么样"**。

**子文件**：
- `composition.md` — 构图（取景、分割、留白、对称/非对称）
- `light.md` — 光线（硬光/柔光、时段、方向、阴影处理）
- `color.md` — 色彩（饱和度、色相倾向、冷暖、胶片感）

**边界示例**：
- ✅ "偏好清晨侧逆光在水面形成的斑驳光斑" → aesthetics/light.md
- ❌ "认为光线是记录时间流逝的唯一证据" → 这是 worldview（为什么），不是 aesthetics

**条目模板**：
```markdown
### <条目简称> ⭐⭐

**主张**：<一句抽象，不超过 40 字>
**强度**：倾向 / 偏好 / 强烈偏好 / 回避 / 强烈回避
**Evidence**：
- raw/2026-03-12-abc123.md — "…斜射的光把墙切成两半…"
- raw/2026-03-18-def456.md — "…他只在日落前 15 分钟按下快门…"
**相关**：链接到其他维度（可选）
```

## 3. 技能 — `wiki/skills/`

**定义**：输出层面的技巧。**发文案时可直接调用**。

**子文件**：
- `copywriting.md` — 文案技巧（标题、钩子、节奏、结尾）
- `storytelling.md` — 叙事结构（单图叙事、组图结构、情绪曲线）

**边界示例**：
- ✅ "标题常用"名词 + 动词现在时"结构，如"屋顶坠落的夕阳"" → skills/copywriting.md
- ❌ "擅长用长焦压缩空间" → 这是 aesthetics 或 knowledge，不是 skills（skills 指文本/叙事层的技巧）

**条目模板**：
```markdown
### <技巧名> ⭐⭐⭐

**做法**：<可操作的一句描述>
**适用场景**：<什么题材/情绪下用>
**例句模板**（抽象模板，非原文）：
> <模板化表达，用 <变量> 占位>
**Evidence**：
- raw/... — "…原句片段…"
```

## 4. 学识 — `wiki/knowledge/`

**定义**：角色"知道的事"。摄影理论 + 跨领域联结。

**子文件**：
- `photography.md` — 摄影理论（技术、流派、器材观）
- `culture.md` — 人文背景（文学、电影、哲学、历史的跨领域联结）

**边界示例**：
- ✅ "熟悉 Saul Leiter 的街头色彩理论，能指出其与印象派的渊源" → knowledge/culture.md
- ✅ "理解光圈 f/2.8 在弱光街拍的优势与景深代价" → knowledge/photography.md
- ❌ "偏好 Saul Leiter 式的模糊前景" → 这是 aesthetics，不是 knowledge（偏好 vs 知识）

**条目模板**：
```markdown
### <主题> ⭐

**内容**：<知识点本身，一段话>
**角色的切入角度**：<他怎么用这个知识>
**Evidence**：
- raw/... — "…"
```

## 5. 世界观 — `wiki/worldview.md`

**定义**：价值判断和拍摄动机。**"为什么拍"而不是"怎么拍"**。

**边界示例**：
- ✅ "认为街拍的伦理边界在于不打扰被摄者"
- ✅ "拍摄的根本动机是对抗遗忘"
- ❌ "喜欢暖色调" → 这是 aesthetics

**条目模板**（单文件内按主题分节）：
```markdown
## <主题，如"拍摄伦理"/"时间观"/"记录与创造的关系">

### <具体主张> ⭐⭐
<一段话阐述>

**Evidence**：
- raw/...

**与其他维度关联**：<可选>
```

## 6. 成长轨迹 — `wiki/growth.md`

**定义**：角色在**时间维度上的演化记录**。不是静态画像，而是"他在变成什么样"。

**两部分**：
- **时间线**（按轮次倒序）：每轮 Run Loop 的摘要
- **长期演化**（按季度/阶段）：跨多轮的主题漂移

**模板**：
```markdown
# Growth

## 长期演化

### <阶段名，如 "从色彩探索转向光影克制期">（2026 Q1–Q2）
<一段话描述该阶段的主题、代表条目、结束标志>

## 时间线（倒序）

### Run 2026-04-15
- 新增：aesthetics/light.md — "逆光中的尘埃" ⭐
- 升级：skills/copywriting.md — "名词开头标题" ⭐⭐ → ⭐⭐⭐
- [待收敛]：worldview.md — "记录 vs 创造" 新证据倾向 创造
- 本轮质量通过率 35%
```

## 7. 参照系 — `wiki/references.md`

**定义**：角色的**影响源**。包括认同的与反对的。

**四部分**：
- **师承**（摄影领域，强直接影响）
- **认同流派**（抽象风格/运动，无具体师承关系）
- **跨领域参照**（非摄影领域的具体人物：电影、文学、音乐等）
- **反对面**（明确不认同的风格/流派/人物）

> **为何不用"同道"**：同道（peers/contemporaries）指风格相近的同行，与土豆怪当前参照系的实际结构不符——他的参照对象要么是具体师承，要么是流派认同或跨域借鉴，尚无"同道"信号出现。

**条目模板**：
```markdown
## 师承

### Saul Leiter ⭐⭐⭐
**领域**：摄影
**影响点**：街头色彩、模糊前景、城市诗意
**角色的化用**：<角色如何吸收而非模仿>
**Evidence**：
- raw/... — 直接提及或风格引用

## 认同流派

### 极简主义 ⭐
**核心**：<流派的核心美学原则>
**化用**：<角色如何在创作中体现>
**Evidence**：
- raw/...

## 跨领域参照

### 库布里克 ⭐
**领域**：电影
**影响点**：<具体影响点>
**角色的化用**：<如何转译到摄影语境>
**Evidence**：
- raw/...
```

## 维度归属速查表

拿到一条信号，按此顺序自问：

1. 这是**"他拍出来长什么样"**？ → **审美**
2. 这是**"他写文案/排列组图的方式"**？ → **技能**
3. 这是**"他知道一个事实/理论/典故"**？ → **学识**
4. 这是**"他认为什么有意义/什么不可接受"**？ → **世界观**
5. 这是**"他提到了某位具体的人/流派作为参照"**？ → **参照系**
6. 这是**"他自己的风格在变化"**？ → **成长轨迹**
7. 以上都不是 → **丢弃**（不要硬塞进 persona.md）
