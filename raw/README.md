# raw/ — 原始数据层

> **只追加，永不修改。**

---

## 规则

1. 每一条高赞摄影帖子在本目录下有且只有**一个独立文件**
2. 文件名格式：`YYYY-MM-DD-<post-id>.md`（YYYY-MM-DD 为抓取日期）
3. 文件一旦写入，**永不覆盖、永不删除、永不改写**
4. 这里是"事实层"——wiki/ 是对它的解释，raw/ 本身不作任何提炼

---

## 文件格式

```markdown
---
post_id: <小红书 id>
author: <作者名>
url: <永久链接>
likes: <数字>
captured_at: <抓取时间 ISO8601>
posted_at: <帖子发布时间 ISO8601>
---

<原帖正文，原样保留——包括错别字、emoji、换行>

<若有图片描述可追加在此，标注 [image-description]>
```

---

## 为什么"只追加"？

- raw/ 的可信性依赖于它的**不可变性**。一旦允许修改，就无法区分"真实历史"和"事后调整的历史"。
- 角色的成长可追溯性——任意时刻都能回到某条 wiki 条目的 evidence，验证它是否忠实于原文。
- 避免 wiki/ 中错误的反向污染 raw/。

如果发现某帖抓取有误（例如内容截断），**新增一个带更新说明的新文件**，不要修改旧文件。
