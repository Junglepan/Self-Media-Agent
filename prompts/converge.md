# Converge — 矛盾收敛 Skill 入口

> 矛盾是成长的信号，但积压的矛盾是认知噪音。
> 本 Skill 让 Agent 系统性地处理 wiki/ 中的所有 `[待收敛]` 条目，做出能做的决策，暂缓不该强行决策的。

---

## 第一步 — 扫描所有 `[待收敛]`

读取 wiki/ 下所有文件，找出所有标注了 `[待收敛]` 的条目对。
列出清单，格式：

```
1. wiki/worldview.md — "记录 vs 创造"
   - 观点 A：…（evidence：raw/xxx）
   - 观点 B：…（evidence：raw/yyy）
   - 上次标注时间：（从 growth.md 查）
   - 已积压轮次：N

2. …
```

---

## 第二步 — 逐条裁决

对每一对矛盾，按以下决策树处理：

### 决策树

```
证据数量是否明显倾向一方？（≥2:1）
  └─ 是 → 保留多数方，删除少数方，去掉 [待收敛] 标记
           在被删除条目旁加注：（收敛于 YYYY-MM-DD，原因：evidence 倾向另一方）
  └─ 否 →
      两条是否实际上不矛盾，只是描述角度不同？
        └─ 是 → 合并（见下方"合并操作模板"）
        └─ 否 →
            已积压超过 6 轮？
              └─ 是 → 做出主动决策（选择更符合 persona.md 整体画像的一方）
                       记录"主动决策，另一方存入 growth.md 作为历史参照"
              └─ 否 → 保持 [待收敛]，更新标注为 [待收敛 · 复查 YYYY-MM-DD]
                       （下次复查日期 = 今日 + 30 天）
```

---

### 合并操作模板

**判断"角度不同"的标准**：两条观点同时为真，且不存在"你对则我错"的逻辑对立。例如：
- 条目 A："偏好逆光" / 条目 B："常在侧逆光下拍"——两者不矛盾，都是真的
- 条目 A："回避过度后期" / 条目 B："不干预原始光线层次"——说的是同一件事

**合并步骤**：

1. **写出合并后的 claim**
   - 取两条中共同的核心倾向，用一句话表达
   - 若两条各有独立意义，可写成带从句的一句（"A，且 B"），但整体不超过 25 字
   - 合并后的 claim 必须仍通过五项 Gate

2. **合并 evidence**
   - 保留两条原 evidence，按 `[来源文件] 引用` 格式并列
   - 置信度取两条中的较高值（但不自动升级；升级需符合 Step 6 规则）

3. **删除原两条，写入合并后条目**，去掉 `[待收敛]` 标记

**合并示例**：

```
合并前：
  - claim: "偏好逆光拍摄" [待收敛]
    evidence: raw/abc001.md — "只有逆光的时候才觉得值得按快门"
  - claim: "常选择光线从侧后方入射的位置" [待收敛]
    evidence: raw/xyz015.md — "每次找机位都先找光的方向再找人"

合并后：
  - claim: "以光源方向为机位选择的第一变量，偏爱逆光与侧逆光"
    evidence:
      - raw/abc001.md — "只有逆光的时候才觉得值得按快门"
      - raw/xyz015.md — "每次找机位都先找光的方向再找人"
    confidence: ⭐⭐
```

---

## 第三步 — 执行变更

按裁决结果修改对应 wiki 文件：
- 收敛（保留一方）：删除被淘汰条目，去掉 `[待收敛]` 标记
- 合并：改写为新表述，去掉 `[待收敛]` 标记
- 主动决策：同收敛，但在 `wiki/growth.md` 追加一条"主动决策"记录
- 延期：更新 `[待收敛]` 后面的复查日期

---

## 第四步 — 重写 persona.md

完成所有收敛后，重新读取六个子文件，重写 `wiki/persona.md`（同 `/skill learn` 的 Step 9 逻辑）。

---

## 第五步 — 汇报

```
Converge 完成
────────────────────────────────
扫描到 [待收敛] 条目：N 对
  ├─ 已收敛（证据倾向）：N 对
  ├─ 已合并（角度不同）：N 对
  ├─ 主动决策：N 对
  └─ 延期复查：N 对（下次：YYYY-MM-DD）
persona.md 已更新
────────────────────────────────
```

---

## 原则

- **不强行统一观点**——不能决策时，延期比强行收敛更诚实
- **主动决策必须留痕**——被淘汰的观点进 growth.md，不是彻底删除
- **Converge 不引入新条目**——只处理已有矛盾，不做学习
