# Publish — 发布 Skill 入口

> 创作 → 确认 → 发布。
> 本 Skill 是 `/skill photographer` 的下游——先生成文案，满意后再用这个 Skill 发出去。

---

## 前提

小红书 MCP 服务必须在本地运行，且已登录。
发布前先调用 `check_login_status` 确认，不可用则中止并提示。

---

## 第一步 — 收集发布素材

你需要提供：

| 字段 | 必填 | 限制 | 说明 |
|------|------|------|------|
| 标题 | 是 | 最多 20 字 | 从 `/skill photographer` 的输出选一个，或自定义 |
| 正文 | 是 | 最多 1000 字 | 从 `/skill photographer` 的输出复制 |
| 图片 | 是 | 本地绝对路径或 HTTP 链接，推荐本地 | 至少 1 张 |
| 话题标签 | 推荐 | 数组 | 从 `/skill photographer` 的标签建议中选取 |
| 发布时间 | 否 | ISO8601 | 不填 = 立即发布 |
| 可见范围 | 否 | 公开 / 仅自己 | 默认公开 |

若以上信息不完整，逐项向用户确认，**不得自行补全标题或正文**。

---

## 第二步 — 发布前确认

在调用 `publish_content` 前，向用户展示完整预览：

```
即将发布到小红书
────────────────────────────────
标题：<标题>

正文：
<正文全文>

标签：#xx #xx #xx
图片：<N> 张
发布：<立即 / YYYY-MM-DD HH:MM>
可见：<公开 / 仅自己>
────────────────────────────────
确认发布？(y/n)
```

等待用户确认后再执行，**不得自动发布**。

---

## 第三步 — 调用 `publish_content`

```
publish_content(
  title    = <标题>,
  content  = <正文>,
  images   = [<路径1>, <路径2>, ...],
  tags     = [<标签1>, <标签2>, ...],
  schedule_at = <ISO8601 或省略>,
  visibility  = <"公开" 或省略>
)
```

---

## 第四步 — 汇报结果

**成功**：
```
发布成功
────────────────────────────────
标题：<标题>
发布时间：<立即发布的时间 / 定时时间>
────────────────────────────────
```

**失败**：
- 说明失败原因（标题超长 / 图片路径无效 / 未登录等）
- 给出修正建议，不自动重试

---

## 原则

- **不自动发布**——必须用户明确确认
- **不修改用户提供的标题/正文**——最多提示超限，由用户决定怎么改
- **不绕过登录检查**——未登录时直接中止，不尝试其他方式
- **发布成功不写入 wiki/**——发布是输出动作，不是学习动作
