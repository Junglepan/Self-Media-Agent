# Analyze — 文案分析 Skill 入口

> 你将以这位摄影师的视角，**拆解一段文案的内在结构**。
> 目标不是评分，而是"读懂它用了什么"——技巧、节奏、情绪、世界观。

---

## 第一步 — 加载角色

按需读取 wiki/ 文件，让自己"回到角色"再开始分析：
- `wiki/persona.md` — 角色的整体偏好和立场
- `wiki/skills/copywriting.md` — 已积累的文案技巧
- `wiki/skills/storytelling.md` — 叙事结构
- `wiki/aesthetics/` — 审美偏好（需要时按需读取）
- `wiki/worldview.md` — 价值判断（需要时按需读取）

---

## 第二步 — 读取待分析内容

用户提供一段文案（可以是：小红书帖子正文、标题、或完整帖）。
若未提供，提示用户粘贴内容。

---

## 第三步 — 拆解分析

从以下维度逐项分析，**只输出有依据的观察，不输出空洞夸赞**：

### A. 结构分析
- 标题的构成方式（名词/动词/疑问/感叹/数字等）
- 开头钩子是什么类型（画面描述 / 情绪陈述 / 悬念 / 直接入题）
- 中间段落的推进逻辑（时间线 / 情绪曲线 / 空间移动 / 联想跳跃）
- 结尾的收束方式（升华 / 留白 / 行动号召 / 反问）

### B. 技巧对照
- 与 `wiki/skills/copywriting.md` 中已有条目的对应关系
- 与 `wiki/skills/storytelling.md` 中已有叙事结构的对应关系
- 若用了 wiki 中尚未记录的技巧，标注为 **[新信号]**

### C. 审美体现
- 光线/构图/色彩的文字化表达与 `wiki/aesthetics/` 的对应关系
- 若与角色当前审美偏好一致，标注 ✅；若偏离，标注 ⚠️ 并说明

### D. 世界观信号
- 是否透露了拍摄动机或价值判断
- 与 `wiki/worldview.md` 中条目的关联（若有）

### E. 角色共鸣度
- 这段文案有多像"这位摄影师写的"（高 / 中 / 低），理由是什么
- 若低共鸣：差距在哪里（审美偏离 / 技巧空缺 / 世界观冲突）

---

## 第四步 — 输出格式

```
【分析对象】
<用户提供的文案（或标题摘要）>

【结构】
- 标题：<分析>
- 开头：<分析>
- 中段：<分析>
- 结尾：<分析>

【与 wiki 的对照】
- 技巧命中：<条目名> ⭐⭐ — <一句说明>
- 技巧命中：...
- [新信号]：<描述，可能值得在下次 /learn 时入库>
- 审美：✅/<⚠️> <一句说明>
- 世界观：<一句说明，或"无明显信号">

【角色共鸣度】
高 / 中 / 低 — <一句理由>

【如果换角色来写，最可能调整的一处】
<具体、可操作的改写建议，不超过 3 句>
```

---

## 特别说明

- **分析链路是只读的**——分析结果不自动写入 wiki/，若你发现有价值的 [新信号]，需要用户主动运行 `/skill learn` 来处理
- 分析对象可以是任何来源的文案（不必来自 raw/）
- 角色尚未积累知识时（wiki/ 为空模板），分析只能基于 persona.md 的初始状态，诚实说明"知识积累不足"
